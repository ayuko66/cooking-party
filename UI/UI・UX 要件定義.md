# **UI/UX 要件定義書（みんなでAIクッキング）**

## # 1. 目的

* PCブラウザ / スマホブラウザの双方で直感的に操作できる UI を提供する。
* 初見のユーザーでも短時間で遊べるシンプルさを維持する。
* スプラトゥーン風のポップで楽しいデザインを採用しつつ、アクセシビリティにも配慮する。

---

## # 2. 対象ユーザー

* 年末年始に友人・家族で遊ぶライトゲーマー
* ゲーム慣れしていないユーザー
* 日本語話者のみ

---

## # 3. 対応デバイス・環境

### ■ デバイス

* PC（1024px〜）
* スマホ（360px〜）
* タブレット（768px以上）

### ■ ブラウザ

* Chrome / Edge / Safari / Firefox の最新バージョン

### ■ orientation

* スマホ縦向き最適化
* 横向きでもレイアウトが破綻しないこと

---

## # 4. 画面一覧

| 画面名           | 目的             |
| ------------- | -------------- |
| トップ / 部屋作成・参加 | ゲーム開始導線        |
| ロビー           | プレイヤー集合・準備     |
| 材料入力（カウントダウン） | 入力中心画面         |
| 調理中           | AI処理中のフィードバック  |
| 結果            | 料理表示・盛り上がりポイント |

---

## # 5. UI機能要件（共通）

### ■ 共通ヘッダー

* Room ID
* プレイヤー数
* 背景は濃色、文字は白

### ■ コンテンツ

* スマホでは 1 カラム
* PCでは余白を確保した 1〜2カラム

### ■ 操作性

* 主要ボタンは画面下部 or 中央に大きく配置
* 44px 以上のタップ領域

---

## # 6. レスポンシブ要件

### ■ ブレイクポイント

* ~480px：スマホ小
* 480〜768px：スマホ大〜タブレット
* 768px〜：PC

### ■ フォント

* 本文：14〜16px（スマホ）
* PC：15〜18px
* 見出し：本文の 1.4〜1.8倍

---

## # 7. アクセシビリティ要件

* テキストと背景のコントラスト比は **AA（4.5:1）以上**
* 色に依存しない情報伝達（アイコン・テキスト併記）
* フォーカスリングを残す
* 動きの激しいアニメーションは避ける
* `prefers-reduced-motion` に対応
* ボタンのタップ領域は **44 × 44px 以上**

---

## # 8. 画面別 UX 要件

### ■ トップ / 部屋作成・参加

* 直感的に「作る / 入る」がわかるボタン配置
* 入力エラーはシンプルな日本語で表示

### ■ ロビー

* プレイヤー一覧カード
* HOST バッジを表示
* スマホでも3人程度が一度に見える

### ■ 材料入力（カウントダウン）

* タイマーは中心要素として大きく表示
* 入力欄は太めの枠で見やすく
* 材料一覧はスクロール可能なカードで表示

### ■ 調理中

* 非同期処理が続く場合、数秒以上かかる可能性を明示
* 過度な動きのないシンプルなアニメーション

### ■ 結果

* 料理名・画像を大きく
* 材料バッジを一覧で表示
* もう一度遊ぶボタンを下部に

---

## # 9. スプラトゥーン風デザイン方針

* ビビッド × ダーク背景
* インクしぶき風の背景テクスチャ
* 角丸 UI
* ホバー時にポップなアニメーション
